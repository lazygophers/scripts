# مقدمة

`scripts` هي مجموعة من أدوات كفاءة التطوير — اختصارات سريعة للمهام الشائعة للتطوير والعمليات. مدخلات خفيفة Bash/Python، مع المنطق الرئيسي في `lib/`.

## النقاط المهمة

- **مدخلات خفيفة** : النصوص البرمجية في `bin/` هي مجرد 3 أسطر من اختراق المسار + استيراد؛ كل منطق الأعمال يعيش في `lib/commands/`.
- **مصنفة حسب المجال** : `build` / `file` / `git` / `process` / `misc` / `system`، معزولة بعضها عن بعض.
- **العمليات بالدفعات** : `merge_*` / `push_*` / `switch_branch` / `sync_master` تغطي المستودع الفردي والدفوعات متعددة المستودعات.
- **السلامة أولاً** : الاستبعاد الذاتي لإدارة العمليات، فحوصات نظافة شجرة العمل والتراجع قبل عمليات Git.

## البدء السريع

```bash
./bin/inject            # حقن bin/ في PATH shell
```

أعد تشغيل shell بعد ذلك، ثم اتصل بـ `checkwork` / `merge_canary` / ... من أي دليل.

انظر [النصوص البرمجية](./scripts.md) ومستودع GitHub [lazygophers/scripts](https://github.com/lazygophers/scripts).
