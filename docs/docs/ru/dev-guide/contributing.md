# Участие

- PR должны включать необходимые модульные тесты.
- Новые скрипты следуют двум шагам [« Добавить скрипт »](./add-script.md), поддерживая разделение между лёгким входом и бизнес-логикой.
- Общие возможности помещаются в `lib/{домен}.py`, повторно используемые между командами, избегая дублирования кода в лёгких входах или отдельных командах.
- Операции Git используют стандартный `git`, избегая интерактивных обёрток ; перед автоматизацией обязательны проверки чистоты рабочего дерева и откаты.
