# Введение

`scripts` — это коллекция утилит эффективности разработки — быстрые сокращения для общих задач разработки и операций. Лёгкие входы Bash/Python, с основной логикой в `lib/`.

## Особенности

- **Лёгкие входы** : скрипты в `bin/` — всего 3 строки хака пути + import ; вся бизнес-логика находится в `lib/commands/`.
- **Классификация по доменам** : `build` / `file` / `git` / `process` / `misc` / `system`, взаимно изолированы.
- **Пакетные операции** : `merge_*` / `push_*` / `switch_branch` / `sync_master` покрывают как одиночный репозиторий, так и пакеты мультирепозиториев.
- **Безопасность прежде всего** : автоисключение управления процессами, проверки чистоты рабочего дерева и откат перед операциями Git.

## Быстрый старт

```bash
./bin/inject            # внедрить bin/ в PATH оболочки
```

После перезапуска оболочки вызывайте `checkwork` / `merge_canary` / ... из любого каталога.

См. [Скрипты](./scripts.md) и репозиторий GitHub [lazygophers/scripts](https://github.com/lazygophers/scripts).
